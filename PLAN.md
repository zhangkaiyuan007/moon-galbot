# 赛题要求

主题：触碰现实

让智能进入真实物理世界：看见物体、理解任务、规划动作，并真正完成一次抓取、移动、装配或交互。重点不是"设备动了"，而是 AI 是否完成了**感知-理解-行动-反馈的闭环**。

# 计划

**机器**：Galbot G1

**任务**：听语音指令，把桌上**随机摆放**的多个物体逐个放入框中；抓取过程中允许对物体进行挪动干扰；支持"抓紧一点/往上抓一点"式在线修正（进阶）。

## 动机：为什么重构架构

此前已用 pi0.5 真机微调端到端跑通全套，但发现**框的位置一变，pick 就受影响**——端到端策略在有限微调数据下把场景布局与行为纠缠在一起学了。

本计划把 **grounding（认哪个、在哪）剥离给预训练 VLM**，让 action 策略坍缩成"朝标记伺服"。换来：布局鲁棒、开放词汇、可调试、推理快。代价：失去 pi0.5 的行为先验，须用示教数据配方（扰动示教）补回。

## 架构

```
G1 麦克风(SDK 唤醒+VAD+降噪) ──> Whisper ASR ──> 关键词映射提取物体名
                                                        │
                                                        v
        ┌──────────── 双速运行架构（工作站）────────────┐
        │  VLM 刷新环(低频 1~2Hz)      ACT 控制环(高频)  │
        │  locate-anything 3B          单策略端到端      │
        │  ──> bbox                    pick→move→place   │
        │      │                           ^             │
        │      v      标记位置(共享状态)     │             │
        │  头部相机画标记 ─────────────────┘             │
        └──────────────────────────────────────────────┘
                                                        │
                                    G1 扬声器 <── TTS 回应
```

- **VLM**：locate-anything 3B (nvidia)，工作站，低频在线 1~2 Hz。抗挪动靠它持续重定位。
- **条件化接口**：**视觉标记**。目标物与框用两种不同标记画在**头部相机**图像上；**腕部相机保持原始画面**（保留"怎么抓、何时合爪"的近距离视觉线索）。不改 ACT 网络结构。
- **Action 模型**：**ACT**，单策略端到端 pick→move→place。任务动作近单模态，推理快，训练省心。
- **语音**：G1 内置麦克风，SDK 提供唤醒词+VAD+降噪（无内置 ASR/TTS）。工作站跑 Whisper 转文本、中英关键词映射提物体名；TTS(piper/edge-tts)→16kHz PCM→SDK 播回。
- **在线修正（进阶）**：策略外三件套——①空间类("往上一点")→平移标记；②力度类("抓紧点")→夹爪参数后处理；③记忆→物体名→修正量 JSON 字典。

## 数据配方（核心）

区分两种鲁棒性，分开采：

- **位置泛化（episode 间）**：100% episode 物体+框随机摆放，框每条换位置 → 治"背地图"。
- **抗挪动干扰（episode 内）**：约 1/4~1/3 episode 在接近途中人为挪动物体一次（偶尔 pick 后挪框），遥操作者自然重新接近完成 → ACT 无先验，恢复行为必须在数据里。
- **多物体在场**：桌上摆多个、只抓被标记的那个，目标物轮换 → 否则学成"抓最近的"，语音选物失效。
- 量级：每任务 50~100 条。旧数据离线用 locate-anything 逐帧标注后叠加复用。
- **标记渲染训练/推理必须严格一致**（颜色、线宽、样式）——此类方案头号翻车点。

## 阶段

1. **离线管线**：部署 locate-anything、实测延迟定频率；标记渲染模块；旧数据逐帧重标注。
2. **语音链路**：SDK 麦克风段 → Whisper → 关键词提物体；TTS 回应。参考 `~/GalbotSDK/examples/g1/python/galbot_robot/audio_example.py`。
3. **新数据采集**：按数据配方遥操作采集。
4. **训练与真机迭代**：ACT 训练，双速部署，迭代至稳定 + 抗挪动验收。
5. **在线修正 + 彩排**：修正三件套；唤醒→语音→VLM→执行→反馈全流程彩排。

## 验证

- **VLM**：比赛桌面照片验证全部物体+框检出与延迟。
- **语音**：真机喊指令，验证唤醒→ASR→物体名提取正确率；TTS 可听。
- **策略**（关键验收）：①框换位置 pick 不受影响；②途中挪物体，重新接近成功；③多物体只抓语音指定的。
- **修正**：说"往上抓一点"抓取点上移；同一物体二次指令自动带上次修正量。

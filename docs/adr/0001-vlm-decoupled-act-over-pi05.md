# 0001 — 用 VLM+ACT 解耦架构替代 pi0.5 端到端

- 状态：已接受
- 日期：2026-07-06

## 背景

此前用 pi0.5 真机微调、端到端跑通了本任务（语音 → 抓桌上随机物体 → 放框）。真机中发现一个稳定复现的失败：**框（place 目的地）的位置一变，pick 流程就受影响**——尽管 pick 逻辑上与框无关。

根因：pi0.5 在有限的真机微调数据下，把"场景布局"与"该做的动作"纠缠在一起学，产生了对框位置的虚假关联。任何端到端策略在小数据下都有此倾向，且失败是黑箱、难以定位与修正。

## 决策

弃用 pi0.5 端到端，改为**分层解耦**：

1. **grounding 剥离给预训练 VLM**（locate-anything 3B），低频在线运行输出 bbox。
2. bbox 以**视觉标记**画在头部相机图像上（腕部相机保持原始）作为条件化输入。
3. action 用**轻量 ACT 单策略**端到端 pick→move→place，任务坍缩成"朝标记伺服"。

## 后果

**收益：**
- **grounding 鲁棒**：物体/框在哪由互联网级预训练的检测模型回答，对布局变化天然免疫；框位置污染 pick 的原 bug 从架构上消除。
- **开放词汇**：只要 VLM 认识，语音说什么都能标，无需为每类物体采示教。
- **可调试**：失败时看标记画在哪，立即区分是 VLM 认错还是策略手笨。
- **在线修正有接入点**："往上抓一点"→平移标记；pi0.5 无此接口。
- **快**：ACT 推理比 3B VLA 快一两个数量级，控制频率更高，对挪动反应更敏捷。

**代价（真实 trade-off）：**
- **失去 pi0.5 的行为先验**。pi0.5 在大规模机器人数据上预训练，自带重试/恢复行为；ACT 从零训练无任何先验。因此"抓取途中物体被挪走→重新接近"这种恢复行为**必须**通过扰动示教（约 1/4~1/3 的 episode）显式采入数据，否则不会发生。这是本决策付出的主要成本，也是数据配方的硬性要求。

## 备选方案

- **继续 pi0.5 端到端 + 加数据**：可能缓解框污染，但仍是黑箱、开放词汇弱、无修正接口，且不解决根因（纠缠学习）。
- **diffusion policy 替代 ACT**：任务动作近单模态，DP 的多模态优势用不上，而推理更慢、调参更繁。故选 ACT。

# CONTEXT — Glossary

本项目的通用语言。只放术语定义，不放实现细节或决策记录决策见 `docs/adr/`。

## Visual marker（视觉标记）
把 VLM 输出的 bbox 渲染到**头部相机**图像上的可见标记，用于告诉 action 策略"抓/放到哪个目标"。目标物与框用两种不同样式区分。**腕部相机永不加标记**，以保留抓取时机所需的原始视觉线索。区别于普通图像标注：它是策略的**条件化输入**，不是训练标签。

## Grounding
把语言里的物体指称（"苹果"）对应到图像中具体像素区域的能力。本项目中这一职责从 action 策略**剥离**给 VLM（locate-anything）承担，策略不再需要"认识"物体。

## Marker servoing（朝标记伺服）
策略被剥离 grounding 后坍缩成的行为：不判断"抓哪个"，只学会把末端移向当前标记所在位置并执行抓/放。空间等变、易学。

## 双速架构（dual-rate architecture）
VLM 刷新环（低频，1~2 Hz）与 ACT 控制环（高频）解耦为两个进程。标记位置经共享状态传递，控制环永远读最新值。挪动物体后标记跳新位置的延迟 = 刷新间隔 + 推理耗时。

## 位置泛化 vs 抗挪动干扰（两种不同的鲁棒性）
- **位置泛化（position generalization）**：物体/框在**不同 episode 之间**位置不同。靠"每条随机摆放"获得，治"背地图"。
- **抗挪动干扰（disturbance robustness）**：**同一 episode 内**、抓取过程中物体被挪走后能重新接近完成。**两者互不替代**——位置泛化对抗挪动零贡献，抗挪动须靠扰动示教。

## 扰动示教（disturbance demonstration）
采集时故意在接近途中挪动物体、遥操作者自然重新接近并完成的一类示教 episode（约占 1/4~1/3）。ACT 无行为先验，"重新接近"这种恢复行为必须出现在数据分布里才会被学到。

## 修正记忆（correction memory）
在线修正的持久化：物体名 → 修正量（z 偏移、夹爪力度等）的 JSON 字典。下次语音提到同一物体时自动预加载上次修正。让机器产生记忆的落地形式。

## 目标物 / 框（target object / bin）
- **目标物（target object）**：单条语音指令要抓取的那一个物体。
- **框（bin）**：place 的目的地容器。此前 pi0.5 架构下框位置变化会污染 pick，本项目正是为解耦此污染而重构。
